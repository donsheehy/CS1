Nesting: One Pattern That Shows Up Everywhere

The idea

Nesting is when a thing contains another thing of the same kind.

A box holds smaller boxes, which hold smaller boxes still. There's no fixed limit to how deep it goes, and the same rule applies at every level. Once you learn to spot this pattern, a lot of computer science stops looking like a pile of separate topics and starts looking like the same shape over and over.

Here are twelve places you'll meet it, roughly from the most familiar to the most powerful.

1. Outlines and menus

A document outline has sections, which contain subsections, which contain sub-subsections. A restaurant menu has categories, and a category can have a sub-menu. Each level looks like the level above it, just narrower.

2. Folders and files

A folder contains files — and other folders. Because a folder is itself a kind of file, the container and the thing contained are the same kind of thing. That's why "look through this folder" naturally means "look through everything inside it, including the folders inside it."

3. HTML tags

A tag can contain other tags:

<div>
  <p>Hello <strong>world</strong></p>
</div>

The <strong> sits inside the <p>, which sits inside the <div>. The nesting is the structure of the page.

4. Nested loops

A loop can run inside another loop. For each step of the outer loop, the inner loop runs all the way through:

for each row:
    for each column:
        print the cell

If the outer loop runs n times and the inner runs n times, you do n × n steps — so nesting also shows you where work piles up.

5. Nested conditionals

An if can live inside another if. You ask one question, and depending on the answer you ask another:

if logged_in:
    if is_admin:
        show admin panel

Each decision opens up a smaller set of further decisions.

6. Nested data

Data holds data. A list can contain lists; a dictionary can contain dictionaries:

classroom = [
    {"name": "Ada", "grades": [90, 85]},
    {"name": "Alan", "grades": [88, 92]},
]

Here a list contains dictionaries, and each dictionary contains a list. Real-world data almost always nests like this.

7. Parenthesized expressions

Parentheses nest, and so does the math inside them:

2 * (3 + (4 - 1))

Every expression is either a plain value or an operation on smaller expressions. To evaluate the whole thing, you work from the innermost parentheses outward.

8. Variable scopes

Blocks of code nest, and so do the "scopes" that decide which variables exist where. An inner block can see variables from the block around it, but the outer block can't see inside the inner one — just like a small box can be aware of the room it's in, but the room can't see inside a closed box.

9. Objects containing objects

An object can hold other objects. A Course holds a list of Student objects, and each Student holds a list of Grade objects. The data model nests the same way the data does.

10. Grammars: the rules of a language

The rules that define a language can refer to themselves. A program is a list of statements; a statement can be a loop; and a loop contains more statements. Because the rules loop back on themselves, a small, finite set of rules can describe infinitely many valid programs (or sentences).

11. The call stack

Nesting can happen in time, not just on the page. When function f calls g, and g calls h, the calls are nested: h has to finish before g can finish, which has to finish before f can. The most recently opened call is the first one to close — the innermost box comes out first.

12. Recursion

A function can call itself. Each call is a smaller copy of the same problem nested inside the last one — like opening a set of Russian dolls. Recursion needs two parts:

  • a base case: the smallest doll, which contains nothing more, and
  • a recursive case: "this problem contains a smaller version of the same problem."

This is nesting turned into a technique: the structure and the solution have the exact same shape.

Where all of this leads: trees

Look back at the twelve examples. Almost every one of them is a tree — a top thing that branches into smaller things, which branch into smaller things, down to the pieces that don't split any further.

  • The top folder branches into folders and files.
  • The <div> branches into the tags inside it.
  • 2 * (3 + (4 - 1)) branches into its smaller expressions.

Trees are the shape of nesting, and recursion is the natural way to work with a tree: handle the current level, then do the same thing to each branch. That's why these ideas keep appearing together. Learn to see the tree, and you'll know how to write the code that walks it.

The one thing to remember: nesting lets a small, fixed set of rules manage unlimited complexity — because the same rule works at every level, all the way down.
