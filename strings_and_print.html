Strings and Print

Hello, World!

Here is the first program that anyone ever writes. It is called Hello World.

print("Hello, World!")

Hello, World!

Easy, right?

The program has two parts. First, there is the print function. The one line of code is calling the print function with the input "Hello, World!". This input is called a string. It's a sequence of individual symbols. Each symbol is called a character. The string is wrapped in quotation marks. You can also use apostrophes:

print('Hello, World!')

Hello, World!

Symbols like parentheses, brackets, braces, quotation marks, backticks, etc. that serve to define a chunk of text are called delimiters. Different delimiters have different meaning in Python. See what happens when we leave them out.

print(Hello, World!)

File "<stdin>", line 1
    print(Hello, World!)
                      ^
SyntaxError: invalid syntax

A program crashes if it gets to a point where it cannot continue. Usually, a crashed program gives an error. In this case, we get a SyntaxError. Syntax is the set of rules for how a program can be written. In this case, it cannot recognize the code as a well-formed Python program. If you look carefully, you will see an arrow pointing at the exclamation point. The exclamation point does have a special meaning in Python, maybe that's the only problem. Let's get rid of the exclamation point and see what happens. It will be less exclamatory, but maybe it will "fix" the error.

print(Hello, World)

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'Hello' is not defined

The error changed. Now, we got a NameError. Specifically, it says name 'Hello' is not defined. It seems to think that Hello is the name of something. This is how Python reads programs. Every piece of text is supposed to mean something. If some text is in quotes, it is treated as a string. Other text will be interpreted as names or keywords (words with special meaning in Python).

We can use the = sign to assign a string to the name Hello. While we're at it, let's assign a string to World, too.

Hello = "What's up,"
World = "Bruh."

print(Hello, World)

What's up, Bruh.

In the example above, Hello and World are called variables. We would say that Hello is a string in the same way that we would say "What's up," is a string. We do this even though Hello is really only referring to a string. If some function expects a string, we can use these interchangeably. When a string is written out and delimited by quotes (or apostrophes), it is called a string literal. It literally tells you what the string is.

It is good to think of variables in Python as names. Names refer to things. There is a string "Bruh." and a variable, World, that refers to it.

Variables are case-sensitiive. consider the following tiny change that breaks our program.

Hello = "What's up,"
World = "Bruh."

print(Hello, world)

Traceback (most recent call last):
  File "<stdin>", line 4, in <module>
NameError: name 'world' is not defined. Did you mean: 'World'?

This error message is more helpful. It is guessing our mistake. In this case, it is right. It won't always be right.

Escaping characters.

Suppose we want to give output that is on two different lines. We might try the following.

print("Hello, 
World!")

File "<stdin>", line 1
    print("Hello, 
          ^
SyntaxError: unterminated string literal (detected at line 1)

It says that we started a string literal with "Hello, but we never "terminated" it. It didn't keep looking until it found the end of the string.

To solve this problem, we could employ a little hack such as the following.

print("Hello,")
print("World!")

Hello,
World!

Each call to print starts its output on a new line. Two print statements gives two lines. Great!

So, why did I call this a hack? If our intention was to have the line break as part of the string, we didn't achieve our goal. We only faked it.

The first step is to realize that not all characters that appear in a string are letters or numbers or punctuation marks. There is a special kind of character called a newline. When it gets printed, the newline character will act like you hit return and start a new line. It might seem like our first attempt to split the output across two lines should have had a newline in the string. But, that newline was in conflict with Python's syntax. It happens that there is a cute way to type a newline.

print("Hello, \nWorld!")

Hello, 
World!

The string \n gets turned into a newline. It is called an escape sequence. There are other escape sequences. One important one is the following. Do you see it?

print("Hello, \\n, my escape sequence friend.")

Hello, \n, my escape sequence friend.

In this one, \\ is used to write a single backslash and not treat it as part of an escape sequence. Here are a couple more.

print("\"quotes\" and", '\'apostrophes\'')

"quotes" and 'apostrophes'

You can see it would be hard to have a string literal that contains quotation marks and apostrophes because typing them would end the string.

A closer look at print

The print function has a bit of magic to it. You'll notice that I said the input was a string, but in our later examples, we gave it two strings. It turns out we can give it any number of strings.

print("Hello,", "sweet", "world!")

Hello, sweet world!

It puts a space between each when printing. If we added the spaces to the strings, we would get extra spaces in the result.

print("Hello, ", " sweet ", " world!")

Hello,   sweet   world!

When giving an input to a function, we say that we are passing the input to the function. Here, we are passing three strings. The inputs are also called arguments to the function. It was very important that we separated the arguments with commas. Here is a version without the commas.

a = "Hello"
b = "sweet"
c = "world"
print(a b c)

File "<stdin>", line 4
    print(a b c)
          ^^^
SyntaxError: invalid syntax. Perhaps you forgot a comma?

Every function has an input and an output. What is the output of print? You might think it's the string that got printed, but that's not right. The text getting printed is a side effect of the print function. It actually returns nothing...or a very specific kind of nothing called None. I'll prove it.

print(print("Hello, World!"))

Hello, World!
None

The inner print function is called with "Hello, World!". When it finishes, it returns its output. That output is then passed to the outer print function. The print function returns a special value called None. That's why it prints None.

I have found that this drives students a little crazy at first. If it drives you a little crazy, that's good. Examine that feeling and repeat after me: "Printing is not returning."
