Arithmetic and Evaluation

Basic Arithmetic

One of the simplest ways to use Python is as a calculator. It has all kinds of arithmetic built in. We can add, subtract, multiply, and divide.

print(1234 + 4321)
print(10000 - 1)
print(8 * 7)
print(10 / 3)

5555
9999
56
3.3333333333333335

We use * to multiply and / to divide.

Wait a minute... Is that last answer correct? How did it round the number to get a 5 at then end? We won't get into the explanation now, but instead, we're going recognize that there is something that the computer is doing that is not the way we might do it. The result is surprising.

Being surprised by the computer is the completely normal. You will write code and expect it to work, but it does something else. Surprise! This will be most of the time. It's not a bad thing.

The first task of the learner is to respond well to this kind of surprise. The best response is curiosity. Have confidence that there is a reason for the behavior that you're seeing. You don't have a problem. You have a mystery. Follow the clues. Try to ask good questions.

Exponents and Roots

Raising a number to a power is also a built-in operation in Python. We use ** to indicate the exponent. Below, we use an f-string (see input_and_fstrings) to drop the result right into the printed text.

x = 11
print(f"{x} squared is {x ** 2}.")
print(f"{x} cubed is {x ** 3}.")

11 squared is 121.
11 cubed is 1331.

This can be used to find square roots using the fact that \[\sqrt{x} = x ^{\frac{1}{2}}.\] 

x = 2
print(f"The square root of {x} is {x ** 0.5}.")
print(f"The golden ratio is {(1 + 5 ** 0.5) / 2}")
print(f"This is pretty close to {89/55}")

The square root of 2 is 1.4142135623730951.
The golden ratio is 1.618033988749895
This is pretty close to 1.6181818181818182

Composing Operations

print(1 + 2 * 3)

7

Some people will expect to get 9 instead of 7. The reason is that they first add 1 and 2 to get 3. Then they multiply by 3 to get 9. The mistake is that they have violated the order of operations. Python uses the standard order of operations. Just as in your math class, you will need parentheses to change the order that the operations are performed.

print((1 + 2) * 3)

9

When you see two operations, like in the example above, it really will need to do two operations. In this case, there is an addition operation and a multiplication operations. The parentheses dictate that the addition happens first. The result is then used in the multiplication. Probably, you have seen this a million times. The key for us is to notice that these operations are functions. Each takes two numbers as its arguments and returns a new number. This means that we will be able to use our understanding of arithmetic to also help us with more exotic functions, including ones that we write ourselve0s.

Expressions and Evaluation

In your math studies, you have probably come across the word expression. An expression is a sequence of symbols that can be evaluated. That means, it can be reduced to a single value. In mathematical expressions, this reduces to a number. An example of an expression is the following.

\[(10 + 3) - (2 + 4)\]

This expression evaluates to $7$. There are other expressions that have the same value such as

\[6+1\] or \[\sqrt{49}.\] Even this one: \[7.\]

These are different expressions with the same value. They are different because they have different symbols. They are equivalent (not equal), because they give the same value.

Key Idea: An expression is a sequence of symbols that can be evaluated to produce a value.

Substitution

In code, most expressions also include variables. In the simplest case, the variable itself is an expression. The value in that case is just the value of the variable. More generally, we substitute the variable with its value in order to evaluate.

x = 42

print(x + 2)

44

When assigning a value to a variable, the right hand side is an expression. The variable is assigned the value that we get when we evaluate the right side. For example, in the expression

x = 10 ** 2
y = x + 1

It works like this. The expression 10 ** 2 is evaluated to produce 100. Then, that value is assigned to the variable x. In the next line, we have another assignment. Again, we evaluate the right side. The first step is to substitute the value for x, so the expression becomes 100 + 1. This is evaluated to give a value 101 that is assigned to y.

The idea of substitution is very important. Any time we are trying to give a value, we can substitute an expression that produces that value. We have been doing this throughout. Every time I called print with something other than a string literal, we understood that this substitution is happening. From one perspective, substitution of this type is the key to all computing.

Updating Variables

Using the equals sign for variable assignment can be a little confusing. The most important thing to remember about assignment is this:

  1. The right side of the assignment is evaluated first.
  2. The actual assignment is the last thing that happens.

This is why we can write code like the following.

x = 5
x = x + 1
print(x)

6

The assignment looks like an equation, but it is not. If it were, it would be saying something false. It can never be true that $x$ is equal to $x+1$. When we write code like this, we are updating the value associated to the variable x. After the first line, the value of x is 5. After the second line, the value of x is 6.

Types

When we discussed the print function, we said that the arguments should be strings. Above, we called print but the arguments were numbers. Numbers and strings are different. All of the things you can store in a Python program are called objects. Every object has a type. Roughly speaking, the type tells you what kind of thing it is. It also tells you what you can do with it. For example, you can print a string.

Let's examine some types by using the type function.

print(type(7))

<class 'int'>

For all normal usage, the words class and type mean the same thing in Python. Okay. 7 is of class int. That's short for integer. Integers can be positive or negative (or zero).

print(type("7"))

<class 'str'>

In this case, we put it in quotes, so it's of class str, short for string.

What about fractions?

print(type(7/3))

<class 'float'>

The class float refers to floating point numbers. The point refers to the decimal point and the floating refers to the way it can appear in different places, unlike an integer.

Everything has a type. Let's test this out:

print(type(print))
print(type(int))
print(type(type(int)))

<class 'builtin_function_or_method'>
<class 'type'>
<class 'type'>

Casting

It is possible to turn a number into a string and vice versa. Converting an object of one type into an object of another type is called casting. Often information is lost in the process.

There is a function str that can turn numbers into strings. (It can do much more than this, but this is enough for now.)

x = 4321
y = str(x)
print(f"The type of x is {type(x)},\nand the type of y is {type(y)}.")

The type of x is <class 'int'>,
and the type of y is <class 'str'>.

Now we have a hint about how the print function works with numbers. Whatever argument gets passed to print, it first converts it to a string by calling str. Then, it has a string to print.

What if it is already a string? No problem. See:

s = "abcd"
t = str(s)
print(f"{t=} is of type {type(t)}")

t='abcd' is of type <class 'str'>

Turning a string into an integer uses the int function. You see the pattern, I'm sure, where these functions have the same name as the class they are converting into. That is the standard convention.

x = "-13"
y = int(x)
print(f"The type of x is {type(x)},\nand the type of y is {type(y)}.")

The type of x is <class 'str'>,
and the type of y is <class 'int'>.

Notice that we can use the negative sign. Let's try to do some impossible stuff.

x = "1.2"
y = int(x)
print(f"The type of x is {type(x)},\nand the type of y is {type(y)}.")

Traceback (most recent call last):
  File "<stdin>", line 2, in <module>
ValueError: invalid literal for int() with base 10: '1.2'
x = 1.2
y = int(x)
print(f"The type of x is {type(x)},\nand the type of y is {type(y)}.")

The type of x is <class 'float'>,
and the type of y is <class 'int'>.

Surprise! That worked? How does it turn 1.2 into an integer? Mystery time.

x = 1.2
y = int(x)
print(f"{x=} and {y=}")

x=1.2 and y=1

It rounded. Does it always round to the nearest integer?

x = 1.9
y = int(x)
print(f"{x=} and {y=}")

x=1.9 and y=1

Nope, it rounds down. What about negative numbers?

x = -1.2
y = int(x)
print(f"{x=} and {y=}")

x=-1.2 and y=-1

It rounded the negative numbers up. How should we think about this so that it makes sense? It is really just throwing out everything after the decimal point.

You can turn an integer or a string into a float using, you guessed it float:

x = 100 
print(f"x has type {type(x)} and float(x) has type {type(float(x))}")

x has type <class 'int'> and float(x) has type <class 'float'>

Here is the string to float conversion.

s = "100" 
print(f"s has type {type(s)} and float(s) has type {type(float(s))}")

s has type <class 'str'> and float(s) has type <class 'float'>

Here is one more, because it's awesome.

print(float("inf") + 1)

inf

That's right. I just printed infinity plus one.

Integer Division and Modular Arithmetic

When you first learned division, you likely did not know about decimals. Do you remember how you did division with only integers?

You get the quotient and the remainder. You might think you've grown out of these operations, but in many cases, these are more useful than regular division.

Two operations are integer division and mod. The first uses the double slash notation // and gives the quotient. The mod operation uses the percent sign % and gives the remainder. Here's an example.

a = 10
b = 3

print(f"{a} divided by {b} is {a // b} with remainder {a % b}.")

10 divided by 3 is 3 with remainder 1.

The word mod is a reference to modular arithmetic.

Formatting Numbers

You can modify the way a number gets formatted in an f-string. See below.

sqrt_of_2 = 2 ** 0.5
giga = 2 ** 32
print(f"The square root of 2 is {sqrt_of_2}.")
print(f"The square root of 2 is {sqrt_of_2:.3g}.")
print(f"2 to the 32nd power is {giga}.")
print(f"2 to the 32nd power is {giga:.2e}.")

The square root of 2 is 1.4142135623730951.
The square root of 2 is 1.41.
2 to the 32nd power is 4294967296.
2 to the 32nd power is 4.29e+09.

The second line adds :.3g to indicate that we only want 3 significant figures. The fourth line adds :.2e to give scientific notation with two digits after the decimal point.
