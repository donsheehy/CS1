Input and f-Strings

It's nice to be able to write a program that we can interact with.

Getting Input from the User

So far, every program has been a one-way conversation. The program talks, using print, and you just read along. The input function flips this around. It lets the program ask a question and wait for you to answer.

name = input("What is your name? ")
print("Hello, ", name)

These notes usually add the output automatically. In this case, because it needs input from you, I can't do that. Instead, try it yourself. Type it into VSCode and run it (see python_in_vscode). If you type Boogerface and press enter, you'll see Hello, Boogerface printed out.

Here's what happened, step by step. First, input printed its prompt, the string "What is your name? ", without starting a new line. Then it paused and waited for you to type something and press enter. Whatever you typed became the return value of input, and that value got assigned to name.

Input Is Always a String

Try asking for a number instead of a name.

age = input("How old are you? ")
print(age + 1)

You might expect that typing 20 gives you 21. It doesn't. Run it yourself and you'll get a crash. It would be like running:

print("20" + 1)

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: can only concatenate str (not "int") to str

Python is treating age as a string, even though you typed digits. The word concatenate appears in the error message. The definition is below.

Here's the mystery, unpacked: input always returns a string, no matter what the user types. It has no way of knowing whether you're answering "what's your name" or "how old are you." Every keystroke just becomes characters, and characters are strings.

If we want to do arithmetic with the answer, we have to cast it ourselves (see Casting).

age = input("How old are you? ")
age = int(age)
print(age + 1)

Now typing 20 gives you 21, as expected.

Key Idea: input always returns a string. If you need a number, you must cast it yourself.

Putting Strings Together

Now that you can read a string from the user, you'll often want to weave it into a bigger message. Python lets you join strings end to end using +.

name = "Ada"
print("Hello, " + name + "!")

Hello, Ada!

This is called concatenation. Notice the space after the comma inside "Hello, " and the "!" tacked onto the end. Python doesn't add spaces or punctuation on its own, so you have to build them into the pieces you're joining.

Combine this with input, and you get a personalized message.

name = input("What is your name? ")
print("Nice to meet you, " + name + "!")

Type Don, and out comes Nice to meet you, Don!.

This works, but it's fussy. Every bit of punctuation has to live in exactly the right piece, and every + has to sit in exactly the right spot. Forget one, and you get a SyntaxError, or a message that reads like Nice to meet you,Don!. There's a nicer way, coming up next.

Building Nicer Output with f-strings

An f-string is a string literal with the letter f written right before the opening quote. Anywhere you put curly braces {} inside it, Python evaluates whatever is inside and drops the result into the string.

name = "Ada"
print(f"Hello, {name}!")

Hello, Ada!

Compare that to building the same string with concatenation.

name = "Ada"
print("Hello, " + name + "!")

Hello, Ada!

Both lines print the same thing, but the f-string version doesn't force you to think about where the spaces and punctuation live relative to the quotes.

The curly braces aren't limited to a plain variable. Anything that's an expression works, including arithmetic.

x = 7
print(f"{x} times 2 is {x * 2}.")

7 times 2 is 14.

Combine this with input, and you can read something from the user and immediately drop it into a nicely formatted message.

name = input("What is your name? ")
print(f"Nice to meet you, {name}!")

Type Don, and out comes Nice to meet you, Don!.
